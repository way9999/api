# Cloudflare Temp Email 配置说明

## 1. 修改 openai_register.py 配置

打开 `openai_register.py` 文件，找到配置区域（第 13-17 行），修改以下参数：

```python
# ========== 配置区域 ==========
WORKER_URL = "https://mail.example.com"  # 改为你的 Worker 地址
ADMIN_PASSWORD = "your_admin_password"  # 改为你的管理员密码
EMAIL_DOMAIN = "example.com"  # 改为你的邮箱域名
SITE_PASSWORD = None  # 如果启用了网站密码，填写这里
# =============================
```

### 参数说明：

- **WORKER_URL**: 你部署的 Cloudflare Worker 地址
  - 例如：`https://mail.example.com`
  - 不要在末尾加 `/`

- **ADMIN_PASSWORD**: 你在部署时设置的管理员密码
  - 在 Cloudflare Workers 环境变量中的 `ADMIN_PASSWORD`

- **EMAIL_DOMAIN**: 你配置的邮箱域名
  - 例如：`example.com`
  - 这是你在 Cloudflare Email Routing 中配置的域名

- **SITE_PASSWORD**: 网站访问密码（可选）
  - 如果你启用了 `SITE_PASSWORD` 环境变量，在这里填写
  - 如果没有启用，保持 `None`

## 2. 测试配置

修改完成后，运行测试：

```cmd
python openai_register.py
```

## 3. 运行注册脚本

配置成功后，运行：

```cmd
python start.py
```

## 常见问题

### 1. 状态码 401 - 未授权
- 检查 `ADMIN_PASSWORD` 是否正确
- 检查 `SITE_PASSWORD` 是否需要设置

### 2. 状态码 404 - 未找到
- 检查 `WORKER_URL` 是否正确
- 确保 URL 不包含末尾的 `/`

### 3. 域名错误
- 检查 `EMAIL_DOMAIN` 是否与你的 Worker 配置一致
- 确保域名已在 Cloudflare Email Routing 中配置

## 4. 获取配置信息

如果你不知道这些配置信息，请查看：

1. **Worker URL**: 
   - 登录 Cloudflare Dashboard
   - 进入 Workers & Pages
   - 找到你的 temp email worker
   - 查看 "Preview" 或 "Triggers" 中的域名

2. **Admin Password**:
   - 在 Worker 的 Settings > Variables 中查看
   - 或者查看你的部署配置文件

3. **Email Domain**:
   - 在 Email Routing 设置中查看
   - 或者访问你的 Worker 前端页面，查看可用域名列表
